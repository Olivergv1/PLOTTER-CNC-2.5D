Introducción al CNC

Contexto: ¿Qué es y cómo funciona?
  • Definición: Las siglas significan Control Numérico Computarizado. Es un sistema de automatización donde las máquinas son operadas mediante comandos programados.
  • El Lenguaje G-Code: Son órdenes textuales que la computadora envía. La máquina lee coordenadas (X, Y, Z) y las traduce en movimientos mecánicos precisos.
  • Aplicaciones Reales: Esta tecnología es la base de impresoras 3D, cortadoras láser, cortadoras de agua a presión industriales y grabado.
  • Nuestro Proyecto: Un Plotter CNC de dibujo. Utiliza un bolígrafo físico para trazar líneas vectoriales sobre papel siguiendo el código G.

Hardware y Electrónica

Arquitectura Física del Plotter
  • Cerebro (MCU): Utilizamos el microcontrolador ESP32, elegido por su capacidad dual-core y conectividad Wi-Fi nativa.
  • Potencia (Drivers): Dos controladores DRV8825. Reciben órdenes lógicas y envían pulsos de alto voltaje.
  • Actuadores:
    • 2 Motores Paso a Paso (NEMA).
    • 1 Servomotor SG90 (Eje Z).
  • Estabilidad: Capacitores electrolíticos para filtrar el voltaje.
  • Periféricos: Sistema de iluminación LED (5V) controlado por software.

Ecosistema de Software

Flujo de Trabajo: Diseño → Control → Ejecución
  • El proyecto se divide en tres capas lógicas esenciales:
  • 1. Inkscape (Diseño): Software vectorial donde dibujamos en un lienzo blanco y lo transformamos a coordenadas matemáticas (G-Code).
  • 2. LaserGRBL (Host): Visualización del trayecto y simulación.
  • 3. FluidNC (Firmware): El sistema operativo instalado dentro del ESP32.
  • Archivo YAML: Configuración crítica dentro del chip que define pines y lógica absoluta.

Inkscape: Generación

Creación del Archivo G-Code
  • Configuración del Lienzo: Establecemos un área de trabajo física de 50x50 mm.
  • Lógica de Vectores: Cada línea visual se convierte en coordenadas numéricas (X10 Y20).
  • Extensiones: Usamos plugins para exportar `.gcode` o `.nc`.
  • Desafío Técnico: Calibración del "origen" (0,0) mediante prueba y error.

LaserGRBL: Visualización

Control en Tiempo Real y Post-Procesado
  • Adaptación del Eje Z: Modificamos comandos láser (ON/OFF) a ángulos de servomotor.
  • Control Síncrono: Ejecución directa por cable USB para pruebas.
  • Inyección de Código: Configuración automática de `M8` (LED ON) y `M9` (LED OFF).

FluidNC: Firmware

El Cerebro Embebido en el ESP32
  • Configuración YAML: Define pines GPIO (Drivers, Servo, LED) y Steps/mm.
  • Gestión de Periféricos: Procesa comandos `M8/M9` para el LED (GPIO 23).
  • Servidor Web (WebUI): Carga inalámbrica de archivos `.nc` a LittleFS.
  • Autonomía: El plotter trabaja solo sin depender de la PC.
